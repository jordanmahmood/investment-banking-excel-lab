# 02. Investment Banking Workflow

## Objective

Learn how analysts actually structure a model: assumptions first, then statements, then schedules, then summary output and checks. The point of this module is workflow discipline, not valuation theory.

## Files

- In this module: `TRU_Three_Statement_Workflow_Practice.xlsx`
- In this module: `TRU_Three_Statement_Workflow_Full_Practice.xlsx`
- Answer keys: `../../answer_keys/TRU_Three_Statement_Workflow_Answer.xlsx` and `../../answer_keys/TRU_Three_Statement_Workflow_Full_Answer.xlsx`

Use the standard workbook pair for the primary build. Use the `_Full` pair if you want the extended workbook version with `CF`, `BS`, `Build_Map`, and a live balance check.

## Workbook Map

- Standard workbook: `Assumptions`, `IS`, `Schedules`, `Summary`, `Checks`
- Extended workbook: `Build_Map`, `Assumptions`, `IS`, `Schedules`, `CF`, `BS`, `Summary`, `Checks`

## Useful Skills Covered

- separating assumptions from calculations
- linking forecast drivers through statements
- keeping cash and balance-sheet logic coherent
- structuring a workbook so another analyst can audit it quickly
- using checks before starting valuation work

## Practice Depth

The standard workbook focuses on workflow discipline and core forecast links. The `_Full` workbook carries the same lesson into a larger file where the cash flow statement, balance sheet, ending cash, and balance check work together.

## Completion Standard

- Standard workbook: forecast years populate across `IS` and `Schedules`, and the summary tab is formula-driven.
- Extended workbook: forecast years populate across `IS`, `CF`, `BS`, and `Schedules`, ending cash ties, and the balance check is zero or immaterially close to zero.

## Suggested Time

- first pass: `75-90` minutes
- timed rebuild: `45-60` minutes

## How It Connects Forward

This workbook is the bridge into the DCF. If you cannot build this cleanly, the DCF will turn into a formula chase instead of a valuation exercise.
